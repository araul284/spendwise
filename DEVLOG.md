## Day 5 — 2026-05-10
**Hours worked:**
**What I did:** 
    1. Vercel deploy failure: Missing dependencies
        - Issue: `react-router-dom`, `lucide-react`, `@supabase/supabase-js`, etc. installed locally but not in the `package.json`
        - Cause: Ran `npm install <pkg>` without `--save` on day 1
        - Fix: `npm install react-router-dom lucide-react @supabase/supabase-js --save`
    2. Test file import errors
        - Issue: `import { runAudit } from '../src/lib/auditEngine';` failed
        - Cause: From `src/tests`, `../` already points to `src/`. Extra `src/` made path `src/src/lib` which doesn't exist. 
        - Fix: Changed to `import { runAudit } from '../lib/AuditEngine'`
        - Fix: Changed to `import type { AuditInput } from '../types/index.ts';`
    3. Vitest not recognized
        - Issue: `"test": "vitest run"` script existed, but package wasn't installed
        - Cause: Planned to add testing later and forgot to run `npm install`
        - Fix: `npm install -D vitest` to add to `devDependencies`
    4.  UI changed in results page
        - Added lighter mode palette colors to maintain overall consistency
**What I learned:** 
    - *Dependencies must be in `package.json`, not just `node_modules`*
        Local `npm install` without `--save` works on local machine but breaks when Vercel builds it. When importing, it must be in `dependencies` or `devDependencies`. `node_modules` is gitignored.
    - *Relative imports are relative to the current file*
        `../` = `src/`. To check, count folders of the file and add one `../` per folder level
    - *`package.json` scripts don't install packages*
        Adding `"test": "vitest run"` doesn't auto-install `vitest`. Always install the tool before adding the script.
    - *Tailwind color scales: 50-200 = light, 800-950 = dark*
        Light UIs use low numbers for backgrounds `bg-color-50`, mid numbers for borders `border-color-200`, high numbers for text `text-color-700`
